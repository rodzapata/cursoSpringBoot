Patrones de diseño mas usado con Spring Boot

Los patrones de diseños mas utilizados son:
1. Patrón MVC (Model-View-Controller):
Descripción: Este patrón organiza el código en tres componentes: el Modelo (donde se encuentran las clases de negocio y datos), el Controlador (que maneja las peticiones del cliente) y la Vista (que muestra los datos o devuelve la respuesta al cliente).
Uso en Spring Boot:Spring Boot usa este patrón especialmente en aplicaciones web o APIs REST.
Los controladores están anotados con @Controller o @RestController, los modelos son las entidades anotadas con @Entity, y la vista, si existe, puede ser manejada con tecnologías como Thymeleaf, o en el caso de APIs REST, el JSON o XML generado.

2. Patrón DAO (Data Access Object):
Descripción: El patrón DAO se utiliza para abstraer el acceso a la base de datos.
Ayuda a separar la lógica de negocio del acceso a los datos, lo que hace que la aplicación sea más mantenible.
Uso en Spring Boot: Los DAOs suelen estar anotados con @Repository y usan tecnologías como JPA o Spring Data para interactuar con la base de datos.
Ejemplo: 
            
                @Repository
                public interface UserRepository extends JpaRepository<User, Long> {
                }
3. Patrón DTO (Data Transfer Object):
Descripción: El patrón DTO se utiliza para transferir datos entre diferentes capas de la aplicación.
Sirve para encapsular los datos que deben ser enviados a través de la red o entre componentes, y a menudo se usa para evitar el exceso de información innecesaria. Uso en Spring Boot: Muy común en aplicaciones REST, donde los datos de las entidades se transforman en DTOs antes de enviarlos como respuesta a los clientes.
Ejemplo: 
            
                public class UserDto {
                    private Long id;
                    private String name;
                    private String email;
                }
4. Patrón Singleton:
Descripción: Garantiza que solo exista una instancia de una clase en toda la aplicación.
Uso en Spring Boot:Spring Boot usa este patrón para gestionar beans dentro del contexto de Spring.
Por defecto, los beans en Spring son Singletons, lo que significa que solo hay una instancia de cada bean por contexto de aplicación.
Ejemplo: 
            
                @Service
                public class UserService {
                    // Esta clase será un singleton
                }
5. Patrón Dependency Injection (DI):
Descripción: Es uno de los patrones más importantes y básicos en Spring.
Permite inyectar las dependencias necesarias en una clase desde afuera en lugar de crear dependencias dentro de la propia clase.
Uso en Spring Boot: Spring Boot utiliza inyección de dependencias de manera predeterminada mediante anotaciones como @Autowired, @Service, @Repository , etc.
Ejemplo: 
             
                @Service
                public class UserService {
                
                    private final UserRepository userRepository;
                
                    @Autowired
                    public UserService(UserRepository userRepository) {
                        this.userRepository = userRepository;
                    }
                }
6. Patrón Factory:
Descripción:Crea instancias de objetos sin que el código de la aplicación sepa los detalles de la creación.
Uso en Spring Boot: Spring Boot permite la creación de beans personalizados mediante fábricas (@Bean en clases de configuración),
lo que sigue el patrón de diseño Factory.
Ejemplo: 
             
                @Configuration
                public class AppConfig {
                
                    @Bean
                    public UserService userService() {
                        return new UserServiceImpl();
                    }
                }
7. Patrón Proxy:
Descripción: Proporciona un intermediario o representante que controla el acceso a otro objeto.
Uso en Spring Boot:Este patrón es utilizado por Spring AOP (Aspect-Oriented Programming) para implementar características
transversales como la seguridad, el logging o las transacciones.
Ejemplo: 
             
                @Transactional
                public void someTransactionalMethod() {
                    // Este método será manejado por un proxy de Spring para transacciones
                }
Conclusión:
El patrón MVC es el más utilizado en Spring Boot, especialmente para aplicaciones web y APIs REST.
Sin embargo, Spring Boot es flexible y permite el uso de otros patrones como DAO, DTO, Singleton, Factory, y Dependency Injection,
todos fundamentales en aplicaciones empresariales. La combinación de estos patrones asegura una arquitectura robusta, mantenible y extensible.
